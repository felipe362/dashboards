Escritório Virtual com Agentes de IA para Mercado Livre

Especificação técnica completa para desenvolvimento do zero. Cobre arquitetura, banco de dados, ciclos de dados, código de cada agente, integrações externas e frontend 3D. Inclui versão alternativa com chat LLM sobre os dados da loja.

Arquitetura Geral

O sistema é composto por uma aplicação Next.js 15 hospedada na Vercel, com 6 agentes de IA implementados como API Routes. Nenhum framework de orquestração externo — cada agente é uma rota HTTP chamada pelo Vercel Cron em horários definidos.

Princípio central: Os agentes NUNCA acessam a API do Mercado Livre diretamente durante a análise. Um agente coleta dados do ML e os salva no Supabase. Outros agentes leem do Supabase, consultam o LLM, e devolvem decisões ao Supabase. A execução final no ML é sempre via rota de aprovação separada.

Stack Técnica Completa

Framework

Next.js 15 com App Router e Turbopack. TypeScript em todo o projeto.

Hosting + Cron

Vercel Pro ($20/mês obrigatório para Cron Jobs). O plano gratuito não suporta crons.

Banco de Dados

Supabase (PostgreSQL). Usar supabase-js com service key no servidor para bypass de RLS.

Autenticação

NextAuth.js com Credentials Provider. Sessões JWT. Adapter Supabase para persistir usuários.

LLM — Texto

Anthropic Claude (claude-sonnet-4-5). Usado por SAC, Analista, Anúncios, ADS, Gestor.

LLM — Imagens

OpenAI gpt-image-1. Usado exclusivamente pelo agente Criativo para gerar fotos de produtos.

Realtime

Ably WebSocket. Servidor publica eventos, cliente React se inscreve e atualiza o escritório 3D.

HTTP Client

ky (fetch moderno com retry e timeout). Substitui axios de forma mais leve.

Frontend 3D

Three.js + React Three Fiber (@react-three/fiber) + Drei (@react-three/drei). Modelos GLTF/GLB.

Estilos

Tailwind CSS. shadcn/ui para componentes (botões, modals, toasts). Framer Motion para animações.

O vercel.json define QUANDO cada agente acorda. O Vercel lê esse arquivo a cada deploy e programa os cron jobs automaticamente. Requer plano Pro ($20/mês).

Como o Vercel dispara — protocolo exato

Vercel Platform faz GET para a rota
Envia o header: Authorization: Bearer {CRON_SECRET}. Timeout de 300 segundos para a função responder.
Rota valida o segredo
Se o header não bater, retorna 401. Isso protege de qualquer um chamar a rota manualmente sem autorização.
Rota executa e retorna 200
O Vercel registra o run no painel em Functions → Cron. Falhas (4xx, 5xx) são visíveis no dashboard e podem configurar alertas por email.

Padrão de autenticação em todas as rotas de cron

// Início de TODA rota que pode ser chamada por cron:
export async function GET(req: NextRequest) {
  const isCron = req.headers.get("authorization") ===
    `Bearer ${process.env.CRON_SECRET}`

  if (!isCron) {
    const session = await getServerSession(authOptions)
    if (!session) return NextResponse.json(
      { error: "Unauthorized" }, { status: 401 }
    )
  }
  // ... continua a lógica do agente
}

View consolidada (usada por todos os agentes de análise)

create view view_mlb_completo as
select
  s.*,
  c.preco_concorrente  as top1_preco,
  c.diferenca_pct      as top1_diferenca_pct,
  c.vendedor_nome      as top1_vendedor,
  c.reputacao          as top1_reputacao,
  case
    when s.semaforo = 'vermelho'       then 'revisar_custo'
    when c.diferenca_pct > 5           then 'pode_subir_preco'
    when c.diferenca_pct < -5          then 'risco_preco'
    when s.estoque <= 3               then 'estoque_critico'
    else 'ok'
  end as decisao_sugerida
from mlb_snapshots s
left join mlb_concorrentes c
  on s.mlb_id = c.mlb_id and c.posicao = 1
where s.snapshot_at > now() - interval '2 hours';

Ciclo 1 — Coleta de Dados do Mercado Livre

Quem dispara: Vercel Cron (schedule: "0 * * * *") — todo hora automaticamente.
Vercel Cron 0 * * * *
→
GET /api/agents/analista/run-snapshot
→
ML API (leitura)
→
Supabase upsert
→
Ably notifica
1. Vercel dispara GET /api/agents/analista/run-snapshot
Header: Authorization: Bearer {CRON_SECRET}. A rota verifica o header, abre registro de run no Supabase (status: "running").
2. Rota chama ML API — lista todos os MLBs ativos
GET api.mercadolibre.com/users/{user_id}/items/search?status=active&limit=50 — paginado (50 por vez), repete até pegar todos os anúncios.
3. Para cada MLB: busca detalhes, concorrentes, visitas
3 chamadas por anúncio via rate limiter (máx 3 req/s). Com 200 anúncios = ~600 chamadas = ~3 minutos. Usa rateLimited() para respeitar o limite do ML.
4. Calcula margem líquida (sem IA)
Cruza: preço do ML, custo cadastrado no Supabase, taxa do ML, frete estimado, impostos. Classifica em semáforo verde/amarelo/vermelho. Lógica de negócio pura.
5. Salva snapshot e concorrentes no Supabase (UPSERT)
Não cria novos registros a cada hora — atualiza os existentes. Sempre tem o dado mais recente de cada MLB. Gera alertas automáticos por regras simples.
6. Fecha o run e notifica via Ably
Atualiza agent_runs com status/duração/custos. Publica no canal "escritorio" — o ponto verde do Analista no 3D fica animado por 3 segundos.

Ciclo 2 — Análise com IA

Quem dispara: Vercel Cron (30 min após o snapshot) ou gatilho manual. Os dados já estão no Supabase — nenhuma chamada ao ML aqui.
Vercel Cron 30 * * * *
→
GET /api/agents/anuncios/criar-ordem
→
SELECT Supabase (dados prontos)
→
LLM decide
→
INSERT ordens_agentes
1. Cron dispara o agente de análise
Não há chamada ao ML. O agente vai direto ao Supabase buscar os dados do último snapshot.
2. SELECT na view_mlb_completo
Filtra anúncios com oportunidade: decisao_sugerida IN ('pode_subir_preco', 'risco_preco'). Retorna JSON limpo com preço, margem, concorrente top 1.
3. Para cada candidato: Claude analisa e decide
Envia JSON compacto ao Claude. Recebe decisão em JSON: { acao, novo_preco, justificativa, confianca }. O LLM nunca recebe dados brutos da API.
4. Salva ordem no Supabase com status correto
Verifica modo: se treinamento → status: "pending" (espera humano). Se automático (dia 15+ / taxa ≥ 85%) → status: "auto_pending" (cron executa sozinho).
5. Notifica UI via Ably
Publica evento com número de ordens criadas. Badge de notificação aparece na interface para o operador revisar ou aguardar execução automática.

Ciclo 3 — Execução de Volta para o ML

Dois gatilhos possíveis: (A) Humano clica "Aprovar" na UI — durante o treinamento. (B) Cron processa ordens "auto_pending" — após os 15 dias. O código de execução é idêntico nos dois casos.
Humano aprova UI OU Vercel Cron
→
POST /api/agents/{agente}/aprovar-ordem
→
Supabase (busca ordem + histórico)
→
ML API (PUT/POST)
→
Supabase (atualiza)
→
Ably notifica
1. Gatilho: humano ou cron chama a rota de execução
A rota recebe o ordem_id. Busca a ordem completa no Supabase.
2. Salva histórico para rollback ANTES de executar
INSERT em historico_precos com o valor atual. Se algo der errado ou o operador quiser desfazer, tem como voltar.
3. Executa no Mercado Livre com o token OAuth
Usa o access_token do vendedor (renovado automaticamente se expirado). Chama o endpoint correto: PUT /items para preço/título/imagem, POST /answers para SAC, etc.
4. Atualiza o Supabase com os novos valores

// lib/treinamento.ts
export async function verificarModo(conta_ml: number) {
  const { data } = await supabaseAdmin
    .from("sac_treinamento")
    .select("*")
    .eq("conta_ml", conta_ml)
    .single()

  if (!data) return { modo: "human", diasRodando: 0, taxa: 0 }

  const podeAuto =
    data.total_dias >= 15         // tempo mínimo
    && data.taxa_aprovacao >= 0.85 // 85%+ aprovadas sem edição
    && data.total_respostas >= 50  // mínimo de amostras

  return {
    modo: podeAuto ? "auto" : "human",
    diasRodando: data.total_dias,
    taxa: data.taxa_aprovacao,
  }
}

// Situações que SEMPRE vão para humano, mesmo no modo auto:
export function precisaHumano(texto: string, confianca: number): boolean {
  const palavrasGraves = ["defeito", "devoluç", "reembolso", "problema", "quebrado"]
  const temPalavraGrave = palavrasGraves.some(p => texto.toLowerCase().includes(p))
  return confianca < 0.75 || temPalavraGrave
}

Dias 1–14 (modo humano)

Toda sugestão vai para aprovação humana. O operador aprova, edita ou rejeita. Cada decisão alimenta os contadores. O sistema aprende o padrão.

Dia 15+ (modo automático)

Se taxa ≥ 85% e mínimo de 50 amostras: execução direta. Casos com confiança baixa ou palavras-chave graves ainda vão para humano mesmo no modo auto.

export async function GET(req: NextRequest) {
  // 1. Autenticar (cron ou usuário)

  // 2. Pegar token ML do Supabase (auto-renova se expirado)
  const token = await getMLToken(1)

  // 3. Buscar perguntas pendentes no ML
  const resp = await ky.get(
    `https://api.mercadolibre.com/questions/search?seller_id=${userId}&status=UNANSWERED`,
    { headers: { Authorization: `Bearer ${token}` } }
  ).json()

  for (const q of resp.questions) {
    // 4. Buscar contexto do produto no Supabase (NÃO no ML)
    const produto = await supabaseAdmin.from("mlb_snapshots")
      .select("titulo, preco, estoque").eq("mlb_id", q.item_id).single()

    // 5. Claude gera resposta
    const ai = await anthropic.messages.create({
      model: "claude-sonnet-4-5", max_tokens: 300,
      system: `Você é o SAC de uma loja no Mercado Livre.
Responda perguntas de compradores de forma clara e educada.
Máximo 2 frases. Não invente informações.`,
      messages: [{
        role: "user",
        content: `Pergunta: "${q.text}"\nProduto: ${produto.titulo}\nPreço: R$ ${produto.preco}\nEstoque: ${produto.estoque}`
      }]
    })
    const resposta = ai.content[0].text
    const confianca = calcularConfianca(resposta, q.text)

    // 6. Decidir: auto ou espera humano
    const modo = await verificarModo(1)

    if (modo.modo === "auto" && !precisaHumano(q.text, confianca)) {
      // EXECUÇÃO DIRETA — envia para o ML agora
      await ky.post("https://api.mercadolibre.com/answers", {
        headers: { Authorization: `Bearer ${token}` },
        json: { question_id: q.id, text: resposta }
      })
      await registrarFeedback(1, true)
    } else {
      // FILA HUMANA — salva pendente
      await supabaseAdmin.from("sac_respostas").insert({
        pergunta_ml_id: q.id, resposta_sugerida: resposta,
        status: "pending_approval"
      })
      await ablyServer.channels.get("escritorio")
        .publish("sac:pendente", { count: 1 })
    }
  }
}

// lib/ml-api.ts
async function getMLToken(conta = 1): Promise<string> {
  const { data } = await supabaseAdmin
    .from("ml_tokens").select("*").eq("conta", conta).single()

  // Renova se expira em menos de 10 minutos
  if (new Date(data.expires_at) < new Date(Date.now() + 600_000)) {
    const novo = await ky.post("https://api.mercadolibre.com/oauth/token", {
      json: {
        grant_type: "refresh_token",
        client_id: process.env.ML_CLIENT_ID,
        client_secret: process.env.ML_CLIENT_SECRET,
        refresh_token: data.refresh_token,
      }
    }).json<any>()

    await supabaseAdmin.from("ml_tokens").update({
      access_token:  novo.access_token,
      refresh_token: novo.refresh_token,
      expires_at:    new Date(Date.now() + novo.expires_in * 1000),
    }).eq("conta", conta)

    return novo.access_token
  }

  return data.access_token
}

// lib/llm.ts
export async function callLLM({ provider = "anthropic", system, user, maxTokens = 500 }) {

  if (provider === "anthropic") {
    const res = await anthropic.messages.create({
      model: "claude-sonnet-4-5", max_tokens: maxTokens,
      system, messages: [{ role: "user", content: user }]
    })
    const text = res.content[0].text
    return {
      content: parseJSON(text),   // tenta parsear JSON, retorna texto se falhar
      usage: {
        input: res.usage.input_tokens,
        output: res.usage.output_tokens,
        custo_usd: (res.usage.input_tokens * 0.000003) + (res.usage.output_tokens * 0.000015)
      }
    }
  }

  if (provider === "openai-text") {
    const res = await openai.chat.completions.create({
      model: "gpt-4o-mini", max_tokens: maxTokens,
      response_format: { type: "json_object" },
      messages: [{ role: "system", content: system }, { role: "user", content: user }]
    })
    return { content: JSON.parse(res.choices[0].message.content!), usage: {} }
  }
}

    anuncios: "idle", ads: "idle", gestor: "idle",
  })

  useEffect(() => {
    const channel = ably.channels.get("escritorio")
    channel.subscribe("agente:status", (msg) => {
      setStatus(prev => ({ ...prev, [msg.data.agente]: msg.data.status }))

    return () => channel.unsubscribe()
  }, [])

// components/chat-assistente.tsx
export function ChatAssistente() {
  const [msgs, setMsgs] = useState([])
  const [input, setInput] = useState("")
  const [loading, setLoading] = useState(false)

  async function enviar() {
    if (!input.trim()) return
    const novaMsgs = [...msgs, { role: "user", content: input }]
    setMsgs(novaMsgs); setInput(""); setLoading(true)

    const res = await fetch("/api/chat", {
      method: "POST",
      body: JSON.stringify({ pergunta: input, historico: msgs })
    })

    // Streaming response
    const reader = res.body!.getReader()
    let resposta = ""
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      resposta += new TextDecoder().decode(value)
      setMsgs([...novaMsgs, { role: "assistant", content: resposta }])
    }
    setLoading(false)
  }

  return (
    <div className="chat-container">
      {msgs.map((m, i) => <MessageBubble key={i} msg={m} />)}
      <input value={input} onChange setInput(e.target.value)}
        onKeyDown={e => e.key === "Enter" && enviar()}
        placeholder="Pergunte sobre sua loja..." />
    </div>
  )
}
